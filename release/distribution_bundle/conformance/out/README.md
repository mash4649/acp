# Conformance outputs

This directory holds run artifacts and **v1 audit evidence** checked into the bundle.

## Versioned audit evidence (tracked in git)

- `v1-evidence.public_release_check.txt` — full stdout/stderr from `./scripts/public_release_check.sh`
- `v1-evidence.conformance_selftest.txt` — full stdout/stderr from `./scripts/conformance_selftest.sh` (after deps install; see below)
- `v1-evidence.required_external.txt` — full stdout/stderr from `./scripts/external_release_gate.sh` with bundled `acp_harness_runner.sh`
- `v1-evidence.security_ops_minimum.txt` — full stdout/stderr from `./scripts/security_ops_minimum.sh all`
- `release-gate/` — structured report and summary for `required-external`
- `security/` — lock fingerprint, minimal SBOM, vuln-scan output

To regenerate all of the above in one pass (requires network for `pip install`):

```bash
./scripts/verify_v1_bundle.sh
```

To run `conformance_selftest.sh` on a clean checkout without manually installing first:

```bash
ACP_AUTO_INSTALL_CONFORMANCE_DEPS=1 ./scripts/conformance_selftest.sh
```

## Ephemeral / local-only (gitignored)

- `run-request.phase1.json` — regenerated by `conformance.sh` on each run
- `report.phase1.json` — same (when using default report path)

These are not normative protocol content.
