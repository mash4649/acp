# ACP Public Editorial Baseline

## Public definition
- ACP is not an execution framework.
- ACP is not a transport protocol.
- ACP is not a payment rail.
- ACP is an accountability layer for delegated AI work.
- prompt is not contract.
- claim is not evidence.
- verification is not settlement.
- append-only causal event history is required.
- policy evaluation must be replayable.
- narrow core, strong invariants, rich companion/profile.

## Core artifact baseline (15)
- agreement_v1
- revision_v1
- event_v1
- evidence_pack_v1
- proof_artifact_v1
- verifier_descriptor_v1
- proof_finality_v1

- freeze_record_v1

## Non-goal baseline
- Do not sell ACP as replacement for MCP/A2A/workflow/payment rails.
- Do not merge evidence/verification/settlement/dispute semantics.
- Do not redefine Core as 7 artifacts.
