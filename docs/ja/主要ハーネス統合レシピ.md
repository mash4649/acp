# 主要ハーネス統合レシピ

このページは、ACP を主要なハーネスとどう接続するかを、同じ型で整理したものです。

どのレシピも次の 3 点で統一します。

| 項目 | 記録する内容 |
| --- | --- |
| 役割分担 | どのツールが実行し、どのツールが説明責任を持つか |
| 実行点 | 実際にワークフローを動かすコマンドや操作 |
| 証跡 | report / artifact / review note をどこに残すか |

## 1) Claude Code

### 役割分担
- Claude Code は対話型の実装セッションを進める。
- ACP は合意境界、revision、証跡、verification report を残す。

### 実行点
- Claude Code でタスクを進める。
- セッション後にリポジトリルートから ACP の conformance コマンドを実行する。

```bash
./scripts/conformance.sh run --reference
```

### 証跡
- 最小再現用の `examples/minimal-task/`
- 生成された verification report を置く `conformance/out/`
- 人間向けの説明を残す事例集やメモ

### レシピ
1. 1 つのスコープに絞る。
2. Claude Code の prompt と ACP の contract を分ける。
3. 結果を `agreement_v1` / `revision_v1` / `verification_report_v1` に落とし込む。

## 2) Codex

### 役割分担
- Codex は実装ループを回す。
- ACP は何を委任し、何が変わり、何が検証されたかを記録する。

### 実行点
- Codex でリポジトリを編集し、最小限の検証を回す。
- 最後に ACP の reference run で締める。

```bash
./scripts/conformance.sh run --reference
```

### 証跡
- ターミナル上のコマンド出力
- `conformance/out/` の verification report
- 受理または改訂された境界の短い説明

### レシピ
1. Codex に 1 つの bounded change を任せる。
2. 変更を `revision_v1` に記録する。
3. settlement や handoff を主張する前に ACP で境界を検証する。

## 3) Cursor

### 役割分担
- Cursor は編集用ワークスペース。
- ACP は編集の周りにある説明責任の記録。

### 実行点
- Cursor でファイルを編集する。
- 同じ checkout に接続した terminal から ACP コマンドを実行する。

```bash
./scripts/conformance.sh run --reference
```

### 証跡
- working tree 上の編集済みファイル
- `conformance/out/` の verification report
- なぜその編集が受理されたかを示す短い note

### レシピ
1. Cursor で最小限の編集を行う。
2. contract artifact と code diff を分ける。
3. 編集が終わったら verification report を添付する。

## 4) Antigravity

### 役割分担
- Antigravity は agent runtime / orchestration layer。
- ACP は runtime の外側にある説明責任の境界。

### 実行点
- Antigravity で agent flow を実行する。
- その flow の外側に ACP の verification を置く。

```bash
./scripts/conformance.sh run --reference
```

### 証跡
- agent output または handoff summary
- `conformance/out/` の ACP report
- revision / evidence / settlement intent に関する別メモ

### レシピ
1. runtime の message stream を ACP contract に混ぜない。
2. 重要な境界変更だけを ACP artifact に変換する。
3. ACP report を proof layer として扱い、orchestration layer とは分離する。

## 早見表

| ハーネス | 向いている使い方 | ACP の境界 |
| --- | --- | --- |
| Claude Code | 対話型の実装 | contract / evidence / verification は ACP に置く |
| Codex | 短い実装ループ | code change と proof を分ける |
| Cursor | 直接編集の作業 | 編集面と説明責任の軌跡を分ける |
| Antigravity | agent runtime / orchestration | runtime は ACP の外に置く |

## なぜ重要か

どのハーネスでも同じコマンドで検証できます。

```bash
./scripts/conformance.sh run --reference
```

ACP の要点は、実行するツールと説明するシステムを分けることです。
