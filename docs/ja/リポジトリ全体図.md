# 初見向けリポジトリ全体図

このページは、どこを編集し、どこを読むのかを一目で分かるようにするための案内です。

## 1行で覚える

- `docs/` = 人が読む説明書
- `assets/` = 編集可能な公開素材
- `release/` = 固定された配布物

## 何をどこで直すか

| 変えたいもの | 触る場所 | 補足 |
| --- | --- | --- |
| 正本となる説明、方針、解説 | `docs/` | 人向けドキュメントの正本です。 |
| ローンチ文言、FAQ、デモ台本、スキーマテンプレート | `assets/` | 公開向け素材の編集元です。 |
| 配布済みバンドルの中身 | `release/distribution_bundle/` | 生成物として扱い、ここを主編集先にしません。 |
| プロトコルスキーマ、ベクタ | `schemas/` と `conformance/` | プロトコルと検証の正本です。 |
| 実行可能なサンプル | `examples/` | クイックスタートやサンプルフローに使います。 |

## 読む順番のおすすめ

1. `docs/README.md`
2. `docs/ja/README.md` または `docs/en/README.md`
3. `docs/ja/リポジトリ全体図.md` または `docs/en/repository_map.md`
4. `assets/README.md`
5. `release/distribution_bundle/README.md`

## すぐ分かる見取り図

- 公開文書を書くなら `docs/`
- メッセージやデモ素材を書くなら `assets/`
- 配布物を確認するなら `release/distribution_bundle/`
- 挙動を検証するなら `conformance/` と `scripts/`

## このページを置いた理由

同じ話題が複数の場所に分散すると、初見の人は「どこが正本か」で迷います。
このページは、その迷いを先に消すための入口です。
