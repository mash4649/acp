# ACP 導入事例集

このページは、ACP の匿名化した導入事例を短くまとめたものです。

## 採用
- 採用チームが `agreement_v1` / `revision_v1` / `verification_report_v1` を使い、候補者レビュー・最終承認・事後監査の境界を分けた。
- 結果: 引き継ぎ説明がしやすくなり、レビュー証跡を契約境界に紐付けやすくなった。

## 外注制作
- 小規模代理店案件で `agreement_v1` / `delegation_edge_v1` / `settlement_intent_v1` / `verification_report_v1` を用い、依頼内容と納品改訂を分離した。
- 結果: 改訂範囲と支払意図を複数の受け渡しをまたいで追跡しやすくなった。

## 調査・検証
- 調査ワークフローで `agreement_v1` / `evidence_pack_v1` / `verification_report_v1` を使い、主張を証拠に結び付けた。
- 結果: チャットログを再構成しなくても、レビュー時に結論までの経路を追いやすくなった。

## マーケティング
- キャンペーン運用で `agreement_v1` / `event_v1` / `settlement_intent_v1` を使い、公開承認と後続アクションの責任境界を記録した。
- 結果: 変更後のインシデントレビューが速くなり、承認者の境界が明確になった。

## 社内マルチエージェント
- 社内エージェント連携で `agreement_v1` / `delegation_edge_v1` / `resource_reservation_v1` / `freeze_record_v1` を使い、子エージェントの作業を制約した。
- 結果: 問題のある分岐だけを freeze しつつ、親の監査経路を失わずに済んだ。

## 注記
- すべての事例は匿名化してあり、短く要点だけを示しています。
- これは参考パターンであり、万能な運用手順ではありません。
