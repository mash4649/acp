# ACP 10 分価値体験

このガイドは、ACP の価値をリポジトリ上で最短で体験するための手順です。

範囲は意図的に絞っています。

- 1 コマンド
- 1 回の成功確認
- 1 件の失敗時 FAQ

## 1) 1 コマンドで実行する

リポジトリルートで次を実行します。

```bash
./scripts/conformance.sh run --reference
```

このコマンドで行うこと:

- reference runner を使って実行する
- contract boundary を検証する
- `conformance/out/` に verification report を出力する

## 2) 成功を確認する

コマンド実行後、出力ディレクトリを確認します。

```bash
ls conformance/out
```

次のような report ファイルが見えれば成功です。

- `report.phase1.json`
- または、現在の profile で生成された `report.*.json`

成功条件:

- コマンドが `0` で終了する
- `conformance/out/` に report ファイルがある
- report に pass 状態と failed summary がない

必要なら report を開き、実行モードと summary の内容を確認してください。

## 3) 失敗時 FAQ

### `Missing required command` と出る

不足しているツールを先に入れてください。

- conformance スクリプトは `jq` を必要とします
- macOS では package manager から入れられます

### report が出る前に失敗する

リポジトリ依存のセットアップが済んでいるか確認してください。

```bash
./scripts/install_conformance_deps.sh
```

その後、もう一度 reference コマンドを実行します。

### report はあるが実行が失敗している

report を開いて次を確認します。

- `status`
- `summary.failed`
- `results`

pass していない場合、原因はたいてい fixture かローカル環境です。

### 実行と説明責任を分けて見たい

ACP では次のように捉えます。

- 実行 = ワークフローを動かすコマンド
- 説明責任 = 何が起きたかを証明する report

この分離が、10 分価値体験の核心です。

## 覚えておくこと

- 1 コマンド実行する。
- 1 つの report を確認する。
- 実行と説明責任を分離する。
