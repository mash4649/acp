# ACP Lite

ACP Lite は、企業向けの重さを避けながら、説明責任だけは残したい小規模チーム向けの最小導線です。

ACP の境界はそのまま維持します。

- 実行 = 作業を動かすコマンド
- 説明責任 = 何が起きたかを証明する report

一方で、余分な重さは外します。

- day 1 ではガバナンス一式を必須にしない
- 複数ハーネスの統合レシピは追わない
- 広い普及プログラムは後回しにする

## 向いている人

- 少人数の社内チーム
- 1 つのリポジトリを管理する担当者
- 証跡を残したい個人メンテナ
- スコープが明確な小さなパイロット

## 最小フロー

### 1) 最小のタスクを 1 つ用意する

まずは境界がわかる最小バンドルを使います。

- `examples/minimal-task/`
- 1 リポジトリ
- 1 スコープ
- 1 つの責任者

### 2) 1 コマンドで実行する

リポジトリルートで次を実行します。

```bash
./scripts/conformance.sh run --reference
```

### 3) 1 つの report を確認する

出力ディレクトリを確認します。

```bash
ls conformance/out
```

成功条件:

- コマンドが `0` で終了する
- `conformance/out/` に report がある
- report が pass になっている

## ACP Lite で残すもの

| 残すもの | 理由 |
| --- | --- |
| 実行と説明責任の分離 | ACP の中核価値 |
| `conformance.sh` の reference run | 再現可能な証明点 |
| `conformance/out/` の report | 後から確認できる証跡 |
| 最小スコープ | 小規模チームでも始めやすい |

## ACP Lite で外すもの

| 外すもの | 理由 |
| --- | --- |
| 完全なガバナンス一式 | 初回導入には重い |
| 複数ハーネスのカタログ | 最初の成功には不要 |
| 企業全体の展開プログラム | 後段でよい |
| 広いユースケース集 | まず 1 つの成功経路を作る |

## 企業版との違い

| 項目 | Enterprise ACP | ACP Lite |
| --- | --- | --- |
| 対象 | 組織全体の展開 | 小規模チーム / 単独リポジトリ |
| 準備 | 完全な文書とガバナンス | 最小の文書と最小の手順 |
| 証明 | 複数用途の採用経路 | 1 コマンド、1 レポート |
| 目的 | チーム横断で説明責任を標準化 | まず早く価値を示す |

## 失敗時の見方

- report が出る前に失敗したら、conformance 依存のセットアップを確認します。
- report はあるが pass していない場合は、`status` / `summary.failed` / `results` を確認します。
- スコープが大きすぎる場合は、1 人の責任者と 1 つの出力に収まるまで小さくします。

## 要点

ACP Lite は、小さな実案件でも「実行」と「説明責任」を分離できることを素早く示すための入口です。
