# リポジトリ構成診断

この文書は、`acp/` 配下の構成が初見利用者にとって分かりやすいか、また各ディレクトリの責務境界が実際の運用に合っているかを診断するためのものです。

## 結論

現状構成は維持でよいです。大きな再編は不要で、次のタスクでは「責務の明文化」を先に行うのが最適です。

大原則は次のとおりです。

- `docs/` は正本となる説明書
- `assets/` は公開向けコピーやスキーマテンプレートの編集元
- `release/distribution_bundle/` は固定済みの配布成果物
- `examples/`、`schemas/`、`conformance/`、`scripts/`、`sdk/` は編集対象のソース領域

## 責務表

| パス | 目的 | 主な編集者 | 主な利用者 |
| --- | --- | --- | --- |
| `acp/docs/` | 公開ドキュメントの正本 | docs 管理者 | コントリビューター、導入検討者、レビュー担当 |
| `acp/schemas/` | スキーマ本体と説明文 | プロトコル管理者 | バリデータ、SDK 作者、テスト作者 |
| `acp/conformance/` | プロファイル、ベクタ、ケース、出力 | プロトコル管理者と CI | ハーネス作者、リリース検証 |
| `acp/examples/` | 実行可能な例とテンプレート | 例題管理者 | ACP を試す利用者 |
| `acp/scripts/` | 検証、リリース、補助スクリプト | メンテナ | CI、リリース自動化、導入者 |
| `acp/sdk/` | SDK 実装 | SDK 担当 | アプリケーション開発者 |
| `acp/assets/` | ローンチ文言、FAQ、デモ台本、スキーマテンプレート | docs / リリース文面担当 | 公開文書編集、ローンチ準備、配布準備 |
| `acp/release/distribution_bundle/` | リリース用の固定済み配布バンドル | リリース自動化 | バンドル利用者、公開バンドル確認者 |

## 境界確認: `assets/` と `release/distribution_bundle/`

### `assets/`

ここは公開向け素材の編集元です。

- コピー集
- デモ台本
- FAQ スニペット
- 異議対応メモ
- スキーマテンプレートの入力

このディレクトリは頻繁に編集される前提です。

### `release/distribution_bundle/`

ここは配布用の固定成果物です。

- 固定済みの docs
- バンドルされたスキーマと例
- リリース用スクリプト
- conformance 出力
- 再現可能なバンドル状態

ここは編集元ではなく、出力物として扱います。

### 判定

この分離は妥当です。維持します。

2 つを統合しない方がよい理由は、ライフサイクルが違うからです。

- `assets/` はソース素材として頻繁に変わる
- `release/distribution_bundle/` はバンドル再生成のときだけ変わる

## 代案比較

| 案 | 内容 | リスク | 判定 |
| --- | --- | --- | --- |
| 最小変更案 | 現状構成を維持し、責務と導線だけ明文化する | 低 | **採用** |
| 全面再編案 | トップレベルの複数ディレクトリを新しい構成へ移動する | 高 | 今回は見送り |

## 最小変更案を選ぶ理由

- 現状の木構造は、ソース・docs・examples・scripts・release を既に分けている
- 混乱の主因はパス名そのものより、責務の説明不足
- 大規模移動は、責務文書がない段階では参照更新コストが高い

## 影響範囲チェックリスト

次のタスクで確認する項目です。

- ディレクトリ責務ドキュメントの更新
- `assets/` を参照する内部リンクの確認
- `release/distribution_bundle/` を参照する内部リンクの確認
- docs index が正本の文書を指しているかの確認
- release bundle のパスが読み取り専用前提になっているかの確認
- examples が利用者向けクイックスタートで使われているかの確認

## S-2 でやるべきこと

S-2 では、この診断をそのまま実装に渡せるように、ディレクトリ責務を短く明文化します。

- ソースか生成物か
- 編集対象か固定物か
- 正本か配布物か
- 誰が何を編集するか

この段階では、ディレクトリ移動は行わず、参照更新だけに留めます。
