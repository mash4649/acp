# ACP 5分クイックスタート

この手順で、クリーンな checkout からローカル conformance 実行までを約 5 分で確認できます。

## 前提
- `python3` 3.11 以上
- `bash` または `zsh`
- 選択した Python に対して `pip` が利用可能

## 1. conformance 依存関係を入れる

```bash
./scripts/install_conformance_deps.sh
```

バージョン付きの `.tmp/` ディレクトリに参照ハーネス依存をローカル配置し、`requirements-conformance.lock` を優先利用します。

## 2. 環境を確認する

```bash
./scripts/conformance.sh doctor
```

期待する結果:
- コマンドが `0` で終了する
- contract / profile / ローカル依存関係が読めると報告される

## 3. ローカル成果物を準備する

```bash
./scripts/conformance.sh prepare
```

期待する結果:
- `conformance/out/` が作成または更新される
- 対象 profile 用の request / report の雛形が生成される

## 4. mock で実行する

```bash
./scripts/conformance.sh run --mock
```

期待する結果:
- 外部依存なしで実行できる
- `conformance/out/` に最新の mock report と request payload が残る

## 5. reference で実行する

```bash
./scripts/conformance.sh run --reference
```

期待する結果:
- schema ベクタと invariant ケースを repo-local の reference harness が評価する
- チェック対象の baseline では report status が `pass` になる

## よくある失敗

`jsonschema` や `yaml` の import error が出る場合は、`.tmp/conformance-deps` を削除して依存関係を入れ直してください。中断されたインストールや Python バージョン不一致が原因であることが多いです。
