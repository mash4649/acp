# 外部ランナー E2E 統合テストフレームワーク

この文書は、ACP の external runner を end-to-end で検証するための最小フレームワーク設計を定義します。
ここで扱うのは設計であり、runner 契約そのものの変更ではありません。

## 目的

ACP には conformance 契約、reference harness、bundled adapter がすでにあります。
不足しているのは、external runner を black box として繰り返し検証し、証跡を明示的に扱える再現可能な枠組みです。

このフレームワークが答えるべき問いは次の 4 つです。

1. runner は期待した contract 入力を受理したか
2. 期待した report と artifact を生成したか
3. 同じ fixture を再実行しても安定しているか
4. 一時的な値に依存せず evidence を比較できるか

## スコープ

対象:

- 固定 fixture からの runner 呼び出し
- 同一 fixture の再実行
- report と evidence の正規化比較
- 後続監査のための evidence 保持

対象外:

- external runner 契約そのものの変更
- ACP schema の変更
- transport 固有の統合
- 最小フレームワークを超える CI マトリクス拡張

## 最小構成

フレームワークは 3 層で構成します。

### 1. Fixture

fixture は自己完結したテストケースです。最低限、次を含みます。

- input profile
- request payload または request manifest
- 期待される artifact 集合
- 期待される evidence 制約

fixture は deterministic でなければなりません。時刻、path、生成 ID が必要な場合は、比較前に正規化します。

### 2. Runner contract

フレームワークは conformance gate と同じ contract で external runner を呼び出します。
境界は保ちます。

- framework は runner 内部に踏み込みません
- framework が観測するのは入力と出力ファイルだけです
- runner は runtime / transport / adapter を自由に実装できます

### 3. Evidence comparison

比較は 2 段階で行います。

- structural validation: report 形状、必須項目、artifact 存在の確認
- normalized stability check: volatility を除いた出力を rerun 間で比較

run ID、generated_at、一時的な metadata は diff 前に削除します。

## 推奨実行フロー

1. 1 つの fixture を読み込む。
2. external runner を 1 回実行し、出力を収集する。
3. 出力を ACP schema と local report shape で検証する。
4. volatility を含む項目を正規化する。
5. 同じ fixture を 2 回目として再実行する。
6. 2 回目の出力を正規化する。
7. 2 つの正規化済み出力を比較する。
8. pass/fail を記録し、raw evidence bundle を保持する。

## Evidence bundle の内容

evidence bundle には最低限、次を含めます。

- 使用した fixture
- 各実行の raw report
- 正規化比較結果
- pass/fail summary
- runner identifier と execution timestamp

bundle は append-only とし、監査レビューに使える形にします。

## 安定性シグナル

次の差分は flake の候補として扱います。

- 同一 rerun 間での正規化済み report 差分
- 2 回目実行での artifact 欠落
- append-only 項目の順序不安定
- version bump なしの report shape 変更

一時的な値があるだけでは失敗にしません。stable field に volatility が漏れた場合に失敗とします。

## 導入の進め方

まずは 1 fixture, 1 runner target から始めます。
基本形が安定してから、次を追加します。

- 複数 fixture
- 複数 runner 実装
- cross-version 比較
- 長期 regression tracking
