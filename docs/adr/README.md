# ADR Directory

This directory stores Architecture Decision Records for ACP.

## Purpose
- keep important design decisions discoverable
- record context, decision, and consequences in a small, reviewable format
- avoid rewriting long discussions when a decision needs to be reused later

## How to use
1. Copy `template.md` to a new file named with the next sequence number, such as `0001-short-title.md`.
2. Fill in the context, decision, and consequences.
3. Keep the record short and link back to relevant docs or issues.

## Files
- `template.md`

## Notes
- ADRs should be concrete and narrow.
- One ADR should cover one decision.
- If a decision changes later, add a new ADR instead of rewriting history.
